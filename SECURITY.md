# Security Policy

## Reporting Vulnerabilities
- Report vulnerabilities via GitHub Issues labeled `security`.
- Do not disclose publicly until patched.

## Secrets Management
- No secrets in repo. Use `.env` and provide `.env.example`.

## Dependency Review
- Dependencies reviewed monthly.
- Patch minor versions proactively.

## Compliance
- ERP modules validated against ISO-9001 tracker.
- Evidence stored in `docs/audit/` for audit-grade traceability.
