## Commit Convention
- Use audit-grade commit prefixes:
  - feat: New feature/module
  - fix: Bug fix
  - docs: Documentation / README / evidence
  - chore: Miscellaneous / binder updates
  - refactor: Code refactoring
  - test: Adding/modifying tests
  - style: Code style / formatting
  - deploy: Deployment changes

## Scope Tags
- frontend, backend, finance, hr, inventory, journey, compliance, docs

## Pull Requests
- PRs must reference evidence in `docs/audit/evidence.json` if compliance-related.
- Ensure code passes lint/test before submission.
